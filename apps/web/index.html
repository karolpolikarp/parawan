Zamaskowano:
Co maskować (domyślnie wszystko — odznaczone typy zostaną widoczne w wyniku)

Opcjonalna usługa uruchamiana na Twoim komputerze — folder services/ner w repozytorium, start jedną komendą: docker compose up -d. Tekst trafia wyłącznie pod wskazany adres lokalny — nadal nic nie wychodzi do internetu. Gdy usługa nie działa, anonimizacja opiera się na warstwie regex + sumy kontrolne.

  • Wklej tekst lub upuść plik (TXT, DOCX, PDF)
  • Analiza reguły, sumy kontrolne, opcjonalny NER
  • Maskowanie kolorowe znaczniki zamiast danych
  • Gotowe skopiuj bezpieczny tekst

Kod pocztowyXX-XXX[KOD-POCZTOWY]
Data urodzeniadata z kontekstem „ur./urodzony”[DATA-URODZENIA]
Adresul./al./os./pl. + nazwa + numer[ADRES]
Imię i nazwiskosłownik ~200 polskich imion + wyzwalacze („nazywam się”, „Pan/Pani”); opcjonalnie lokalny NER (spaCy)[IMIĘ I NAZWISKO]

Walidacja sum kontrolnych odróżnia prawdziwy PESEL czy NIP od przypadkowego ciągu cyfr (sygnatury akt, numery przepisów) — dlatego fałszywych trafień jest bardzo mało. Strażnik kontekstu pilnuje, żeby „art. 123 456 789” nie został uznany za telefon.

Ograniczenia — przeczytaj

Wykrywanie imion i nazwisk jest heurystyczne: rzadkie nazwisko bez imienia ze słownika i bez wyzwalacza kontekstu może nie zostać zamaskowane. Numery w nietypowych formatach też mogą się prześlizgnąć. Zawsze przejrzyj wynik, zanim go udostępnisz. To narzędzie pomocnicze, a nie gwarancja anonimizacji.
